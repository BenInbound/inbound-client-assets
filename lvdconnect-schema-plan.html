Phase 1: Critical Implementation (Priority 1)

Focus: Immediate business impact and fundamental schema structure

Organization Schema Missing
Priority 1

Implement Organization schema to establish business identity across the platform.

Required Properties:

  • @type: "Organization"
  • name: Company legal name
  • url: Homepage URL
  • logo: URL to company logo (at least 112x112px)
  • description: Company description (50-160 characters)
  • sameAs: Social media profiles
  • foundingDate: Year company was founded
  • areaServed: Geographic areas served

Implementation Location:

Add to site header/homepage as JSON-LD script tag

Expected Impact:

  • Improved Knowledge Panel eligibility
  • Better brand recognition in search results
  • Enhanced social media sharing
BreadcrumbList Schema Partial
Priority 1

Implement breadcrumb navigation schema to improve site navigation visibility in search results.

Required Properties:

  • @type: "BreadcrumbList"
  • itemListElement: Array of breadcrumb items
  • Each item requires: position, name, item (URL)

Implementation Location:

Add to navigation elements on all pages with multiple levels

Current Status:

Partially implemented on main pages. Need to extend to all product/solution pages.

Technical Implementation Guide

JSON-LD Format (Recommended)

All schemas should be implemented using JSON-LD format for maximum compatibility and ease of maintenance.

<script type="application/ld+json"> { "@context": "https://schema.org", "@type": "Organization", "name": "lvdconnect", "url": "https://lvdconnect.no", "logo": "https://lvdconnect.no/logo.png", "description": "Business intelligence and data management platform", "sameAs": [ "https://www.linkedin.com/company/lvdconnect", "https://twitter.com/lvdconnect" ], "foundingDate": "2020", "areaServed": ["NO", "SE", "DK"] } </script>

Implementation Best Practices

  • Use JSON-LD format for all schemas
  • Place schema code in the `` section or immediately before closing ``
  • Ensure all URLs are absolute (include domain)
  • Keep data synchronized with actual page content
  • Use HTTPS URLs for images and external links
  • Validate all schemas with Google's Rich Results Test
  • Test across different devices and browsers
  • Monitor performance in Google Search Console

Common Pitfalls to Avoid

  • Incorrect Data Types: Ensure properties use correct data types (string, date, number, etc.)
  • Missing Required Fields: Always include all required properties for each schema type
  • Relative URLs: Always use absolute URLs in schema markup
  • Outdated Information: Keep schema data synchronized with actual page content
  • Multiple Conflicting Schemas: Avoid marking the same element with conflicting schema types
  • Poor Image Quality: Use high-quality, relevant images meeting minimum size requirements
  • Incomplete Phone Numbers: Include full international phone numbers with country code
  • Missing Breadcrumb URLs: Ensure each breadcrumb item links to valid page
