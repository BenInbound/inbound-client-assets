Publish to Web — Setup Guide

Get set up in 2 steps. After that, just say "publish this" in Cowork and you'll get a shareable client URL instantly.

1
Connect GitHub to Cowork

In Cowork, go to Settings → Developer → Edit Config. This opens a file called claude_desktop_config.json.

Your file probably already has some content in it. You need to add the mcpServers block — don't delete what's already there. The result should look like this:

{
  "preferences": {
    ... (your existing preferences stay here)
  },
  "mcpServers": {
    "github": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-github"],
      "env": {
        "GITHUB_PERSONAL_ACCESS_TOKEN": "ASK_BEN_FOR_TOKEN"
      }
    }
  }
}

In other words: add a comma after your last existing block, then paste the "mcpServers": { ... } section before the final closing }.

Save the file, then restart Cowork.

⚠️ Ask Ben for the token to paste in — don't share it over Slack or email.
2
Install the plugin

Ben will send you a file called inbound-publish.plugin.

In Cowork, go to Settings → Plugins and drag the file onto the install area (or click to browse for it).

That's it — you're done.

How to use it

publish this Publishes the HTML file you just made and gives you a live URL
share this with the client Same thing — triggers the publish flow
/publish Explicit command if you prefer that style
Files are named automatically based on the client and content (e.g. odins-brand-guidelines.html) and live at:
inbound-client-assets.netlify.app/{filename}

The link goes live within about 30 seconds. Nothing is ever named index.html, so you can't overwrite someone else's work.
