FRONT Leadership currently has no JSON-LD schema markup across its key pages. This is a significant opportunity: without schema, AI search engines treat all leadership development companies as interchangeable. Priority 1: Highest Impact

Organization Site-wide Priority 1

This is your company identity for AI engines. When someone asks ChatGPT or Claude "What Norwegian leadership development firms offer executive coaching?", Organization schema is what tells them FRONT exists, what you do, where you operate, and how to cite you. This goes on every page of your site.

Applies to: All pages (added once in site header)

Where in HubSpot: Settings > Website > Pages > Site Header HTML (or in your theme's base.html template)

FAQPage Schema (via FAQ Module) Module Priority 1

This is the single most extractable schema for AI citation. When someone asks ChatGPT "What should I expect from executive coaching?" or "How does team leadership development work?", FAQ schema with pre-packaged Q&A is exactly what AI engines look for. FAQ schema essentially pre-answers common questions in the exact format AI needs.

Applies to: Service pages (In-house programs, Coaching, Team Development, Consulting). Recommend 5–8 FAQ pairs per service page.

Where in HubSpot: Build a reusable FAQ module. Add it to service page templates or individual pages.

Module Configuration (fields.json)
[
  {
    "name": "faq_items",
    "label": "FAQ Items",
    "type": "group",
    "occurrence": {
      "min": 1,
      "max": 50
    },
    "children": [
      {
        "name": "question",
        "label": "Question",
        "type": "text",
        "required": true
      },
      {
        "name": "answer",
        "label": "Answer",
        "type": "richtext",
        "required": true
      }
    ]
  }
]
Module Template (module.html)
{# Visible FAQ content #}
<div class="faq-section">
  {% for item in module.faq_items %}
    {% if item.question and item.answer %}
    <div class="faq-item">
      <h3 class="faq-question">{{ item.question }}</h3>
      <div class="faq-answer">{{ item.answer }}</div>
    </div>
    {% endif %}
  {% endfor %}
</div>

{# Schema markup (invisible) #}
{% if module.faq_items | length > 0 %}
<script type="application/ld+json">
{
  "@context": "https://schema.org",
  "@type": "FAQPage",
  "mainEntity": [
    {% set comma = joiner(",") %}
    {% for item in module.faq_items %}
      {% if item.question and item.answer %}
      {{ comma() }}
      {
        "@type": "Question",
        "name": "{{ item.question | escape }}",
        "acceptedAnswer": {
          "@type": "Answer",
          "text": "{{ item.answer | striptags | escape }}"
        }
      }
      {% endif %}
    {% endfor %}
  ]
}
</script>
{% endif %}

Content tip: AI can draft 5–8 FAQ pairs per page in minutes by analyzing your service page content. A content writer then reviews for accuracy and tone in 30–60 minutes per page. Example FAQs for Coaching service: "What should I expect from a coaching engagement?", "How long does executive coaching take?", "What outcomes can I expect from leadership coaching?", etc.

Applies to: All service pages (6 services: Leadership Development, In-house Programs, Coaching, Team Development, Consulting, and any others)

Priority 2: Important Foundation

WebSite + SearchAction Site-wide Priority 2

Identifies your site as a unified entity and tells AI engines (and Google AI Overviews) that the site has search functionality. This provides structural context for all your pages and can enable sitelinks search box features in AI results.

Applies to: All pages (added once in site header)

Where in HubSpot: Settings > Website > Pages > Site Header HTML

JSON-LD Code
{
  "@context": "https://schema.org",
  "@type": "WebSite",
  "name": "FRONT Leadership",
  "url": "https://www.frontleadership.com/en/",
  "publisher": {
    "@type": "Organization",
    "name": "FRONT Leadership"
  },
  "potentialAction": {
    "@type": "SearchAction",
    "target": {
      "@type": "EntryPoint",
      "urlTemplate": "https://www.frontleadership.com/en/search?q={search_term_string}"
    },
    "query-input": "required name=search_term_string"
  }
}

Note: Only include SearchAction if your site has a working search function. Omit it if not.

WebPage (typed) Template Priority 2

Provides explicit page-level signals. An "AboutPage" tells AI engines "this is the authoritative source for company information." Helps AI distinguish a homepage from a service page or about page.

Applies to: Homepage (WebPage), About/People page (AboutPage), service listing page if you have one (CollectionPage)

Where in HubSpot: Add to the page template or individual page head HTML in Settings > Advanced > Head HTML.

JSON-LD Code (About Page example)
{
  "@context": "https://schema.org",
  "@type": "AboutPage",
  "name": "About FRONT Leadership",
  "description": "Meet the team behind FRONT Leadership. We are a group of experienced leadership development professionals based in Oslo, Norway.",
  "url": "https://www.frontleadership.com/en/menneskene",
  "isPartOf": {
    "@type": "WebSite",
    "url": "https://www.frontleadership.com/en/"
  }
}
BreadcrumbList Template Priority 2

Helps AI engines understand your site's structure and hierarchy. When AI reads your breadcrumbs, it learns that services are nested under the main site, which reinforces your organizational structure in its understanding.

Applies to: Service pages and any pages with a clear navigation hierarchy

Where in HubSpot: Add to service page template using HubL navigation/breadcrumb data

JSON-LD Code (Service page breadcrumb example)
{
  "@context": "https://schema.org",

Priority 3: Valuable Additions

Person One-off Priority 3

E-E-A-T (Experience, Expertise, Authoritativeness, Trustworthiness) matters hugely for AI citation. When AI engines can connect your company to named leaders with visible expertise and credentials, they're more likely to cite you as authoritative. Person schema on your team/people page builds this trust signal.

Applies to: Key team members on the About/People page (consider 3–5 key leaders)

Where in HubSpot: Add to the About/People page in head HTML, or build a reusable Person module if team bios are editable.

JSON-LD Code (Team member example)
{
  "@context": "https://schema.org",
  "@type": "Person",
  "name": "[Founder/Leader Name]",
  "jobTitle": "[Job Title]",
  "worksFor": {
    "@type": "Organization",
    "name": "FRONT Leadership"
  },
  "description": "[2-3 sentence bio emphasizing experience and expertise in leadership development]",
  "knowsAbout": [
    "Leadership Development",
    "Executive Coaching",
    "Organizational Change",
    "Team Dynamics"
  ],
  "sameAs": "https://www.linkedin.com/in/[person]",
  "image": "https://www.frontleadership.com/hubfs/team/[name].jpg"
}

Repeat this block for each key team member, using real names, titles, and LinkedIn URLs from your team page.

LocalBusiness Site-wide Priority 3

Useful for location-based AI queries ("Which leadership development companies are in Oslo?"). Helps FRONT show up when someone in Norway asks for local professional services. Combine this with Organization schema using multiple @type values.

Applies to: Homepage (if physical office location is relevant to your business model)

Where in HubSpot: Site Header HTML or homepage head HTML

Note on content hours: Content hours apply only to FAQ writing. All other schema types (Organization, Service, WebPage, Person, BreadcrumbList, etc.) pull their content directly from what already exists on your site: company name, service descriptions, team bios, page titles, meta descriptions. No new content needs to be written. FAQ content hours assume an AI tool generates a first draft of 5–8 Q&A pairs per service page (takes minutes), then a content writer reviews for accuracy, tone, and completeness (typically 30–60 minutes per page). If FRONT already has FAQ content visible on your pages, these hours drop to zero since the dev just maps existing content into schema.
Note on dev hours: Hours assume AI-assisted development (using Claude or ChatGPT to draft HubL code and JSON-LD), which roughly halves traditional build times. The biggest efficiency win is template-level implementation: the Service schema in your service template takes 1.5 hours to build once, then automatically applies to all 6 service pages. Similarly, BreadcrumbList in the service template covers all service pages with one implementation. This template leverage is why a total of 16–19 dev hours covers 50+ pages. Testing and validation (1.5h) covers running all schema through Google's Rich Results Test and Schema Markup Validator, plus manual spot-checks on key pages to ensure HubL variables are rendering correctly.
