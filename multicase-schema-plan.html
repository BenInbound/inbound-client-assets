FAQPage Module Priority 1

FAQPage schema is the single most extractable type for AI citation. When someone asks Claude or ChatGPT "What are the benefits of using Multicase?" or "How much does Multicase cost?", FAQ schema pre-packages your answers in the exact format AI engines need. This drives higher citation accuracy and visibility.

Applies to: Produkter (Products) pages, possibly home page and service pages (4 pages with 5-8 FAQs per page = 20-32 QA pairs)

Where in HubSpot: Build custom FAQ module with repeater fields for questions/answers. Module auto-generates both HTML and FAQPage schema.

JSON-LD Code (Example for Product/Service Page)
{
  "@context": "https://schema.org",
  "@type": "FAQPage",
  "mainEntity": [
    {
      "@type": "Question",
      "name": "What is Multicase and who is it for?",
      "acceptedAnswer": {
        "@type": "Answer",
        "text": "Multicase is a cloud-based business system designed for retail and e-commerce businesses of all sizes. It's ideal for companies that need order management, inventory control, invoicing, and online store functionality in one integrated solution. Multicase serves both B2B and B2C businesses across Norway."
      }
    },
    {
      "@type": "Question",
      "name": "What modules and integrations does Multicase offer?",
      "acceptedAnswer": {
        "@type": "Answer",
        "text": "Multicase offers flexible module architecture including payment integrations (Vipps), return management, price optimization tools, Google Merchant Center integration, customer portal functionality, and 24SevenOffice accounting integration. You can configure the system with only the modules you need."
      }
    },
    {
      "@type": "Question",
      "name": "How long has Multicase been in business?",
      "acceptedAnswer": {
        "@type": "Answer",
        "text": "Multicase was founded in 1997 and has over 25 years of experience in retail and e-commerce software solutions. The company is based in Sandefjord, Norway, with 11-50 employees."
      }
    },
    {
      "@type": "Question",
      "name": "Can Multicase integrate with my existing accounting software?",
      "acceptedAnswer": {
        "@type": "Answer",
        "text": "Yes. Multicase integrates with 24SevenOffice accounting and bookkeeping software, making it easy to sync orders, invoices, and financial data between systems. This integration is available as a standard module."
      }
    }
  ]
}
Service Template Priority 1

When someone asks "What e-commerce solutions are available in Norway?" or "Who offers online store platforms?", Service schema gives AI engines precise, structured answers with service names, descriptions, and provider information. This drives targeted service-level visibility.

Applies to: Forretningssystem og nettbutikk page and other main service/product pages (2-3 pages)

Priority 2: Important Foundation

WebSite + SearchAction Site-wide Priority 2

Identifies the site as a searchable entity and helps AI engines understand your site structure. This is foundational metadata that ties all pages together and supports AI Overviews features in Google.

Applies to: All pages (site header)

Where in HubSpot: Settings > Website > Pages > Site Header HTML (same location as Organization schema)

JSON-LD Code
{
  "@context": "https://schema.org",
  "@type": "WebSite",
  "name": "Multicase",
  "url": "https://www.multicase.no",
  "publisher": {
    "@type": "Organization",
    "name": "Multicase Norge AS"
  }
}

Note: If the site has a search function, add SearchAction property. If not, use WebSite as shown above.

BlogPosting / Article Template Priority 2

Marks up blog articles with author, publish date, and publisher info. AI engines weigh content authorship and recency heavily. Article schema with clear publication data, named authors, and publisher signals gives AI confidence in content credibility. This supports E-E-A-T (Expertise, Experience, Authoritativeness, Trustworthiness) signals that influence AI citation ranking.

Applies to: All blog posts (artikler section) - approximately 15+ posts

Where in HubSpot: Add to blog post template using HubL variables for title, author, date, image, URL

JSON-LD Code (Template with HubL Variables)
<script type="application/ld+json">
{
  "@context": "https://schema.org",
  "@type": "BlogPosting",
  "headline": "{{ content.name | escape }}",
  "description": "{{ content.meta_description | escape }}",
  "url": "{{ content.absolute_url }}",
  "datePublished": "{{ content.publish_date | datetimeformat('%Y-%m-%d') }}",
  "dateModified": "{{ content.updated | datetimeformat('%Y-%m-%d') }}",
  {% if content.blog_author %}"author": {
    "@type": "Person",
    "name": "{{ content.blog_author.display_name | escape }}"
  },{% endif %}
  "publisher": {
    "@type": "Organization",
    "name": "Multicase Norge AS",
    "logo": {
      "@type": "ImageObject",
      "url": "https://www.multicase.no/hubfs/multicase-logo.svg"
    }
  }{% if content.featured_image %},
  "image": "{{ content.featured_image }}"{% endif %}
}
</script>
BreadcrumbList Template Priority 2

Helps AI engines understand site hierarchy and page relationships. A BreadcrumbList explicitly states: "This page is under Products > Modules" or "This is a Blog Post under Blog". This context helps AI engines cite content more accurately and understand section relationships.

Applies to: All main page templates (product pages, blog posts, etc.)

Where in HubSpot: Add to base template or individual templates using page URL structure and HubL variables

JSON-LD Code (Example for Product Page)
{
  "@context": "https://schema.org",
  "@type": "BreadcrumbList",
  "itemListElement": [
    {
      "@type": "ListItem",
      "position": 1,
      "name": "Home",
      "item": "https://www.multicase.no"
    },
    {
      "@type": "ListItem",
      "position": 2,
      "name": "Produkter",
      "item": "https://www.multicase.no/produkter"
    },
    {
      "@type": "ListItem",
      "position": 3,
      "name": "Forretningssystem og nettbutikk",
      "item": "https://www.multicase.no/produkter/forretningssystem-og-nettbutikk"
    }
  ]
}

Priority 3: Valuable Additions

WebPage Template Priority 3

Provides explicit signals about page purpose and type. A WebPage with @type "AboutPage" or "ContactPage" tells AI engines how to treat each page. This helps AI engines cite the right page type for different queries.

Applies to: Key landing pages (About, Contact, Product overview pages) - 3-4 pages

Where in HubSpot: Can add to specific page templates or use page-level head HTML for one-off pages

JSON-LD Code (Example for About Page)
{
  "@context": "https://schema.org",
  "@type": "AboutPage",
  "name": "About Multicase Norge AS",
  "description": "Learn about Multicase Norge AS, a Norwegian business system and e-commerce solution provider founded in 1997 with 25+ years of experience.",
  "url": "https://www.multicase.no/om-multicase",
  "isPartOf": {
    "@type": "WebSite",
    "url": "https://www.multicase.no"
  }
}
Person (Team Members) One-off Priority 3

Strengthens E-E-A-T signals by connecting content and company information to named experts. When AI engines can identify team members with specific expertise, they're more likely to cite company content as authoritative. Good for team/about pages and potentially blog author pages.

Applies to: About page team section or individual team member pages (if available) - optional based on content visibility

Where in HubSpot: Add to about page using page-level HTML, or create a reusable person module for team pages

JSON-LD Code (Example for Team Member)
{
  "@context": "https://schema.org",
  "@type": "Person",
  "name": "[Team Member Name]",
  "jobTitle": "[Job Title]",
  "worksFor": {
    "@type": "Organization",
    "name": "Multicase Norge AS"
  },
  "description": "[Brief bio describing expertise]",
  "knowsAbout": ["Business Systems", "E-commerce", "Retail Management"]
}

Note on content hours: Content hours apply only to FAQ writing. All other schema types (Organization, Service, BlogPosting, BreadcrumbList, WebSite, WebPage, Person) pull content directly from what already exists on the multicase.no site (company info, service descriptions, blog posts, team info, page titles). No new content needs to be written for these. FAQ content hours assume AI generates a first draft of 5-8 Q&A pairs per service page, then a content writer reviews and edits for accuracy (typically 30-60 minutes per page). If Multicase already has FAQ content visible on its product or service pages, these hours drop to zero.
Note on dev hours: Dev hours assume AI-assisted development, which roughly halves traditional build times. The template-level efficiency is key: most of the work is done once and applies automatically to all pages of that type. For example, adding BlogPosting schema to the blog template covers all 15+ existing blog posts without individual per-page work. BreadcrumbList on the base template covers 50+ pages automatically. Testing and validation (1-2 hours) is included to ensure all schema is syntactically correct and passes Google Rich Results and Schema Markup Validator checks.
