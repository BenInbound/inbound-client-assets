Priority 2: Important Foundation

WebSite (with SearchAction) Site-wide Priority 2

Identifies capa.no as a searchable website and provides AI engines with top-level navigation context. Helps AI understand how to query your site and supports features like sitelinks in Google AI Overviews. If CAPA has a site search function, this schema tells AI engines how to search it.

Applies to: All pages (added once in site header)

Where in HubSpot: Settings > Website > Pages > Site Header HTML (alongside Organization schema)

JSON-LD Code
{
  "@context": "https://schema.org",
  "@type": "WebSite",
  "name": "CAPA Kinoreklame",
  "url": "https://capa.no",
  "publisher": {
    "@type": "Organization",
    "name": "CAPA Kinoreklame AS"
  }
}

Note: Include SearchAction only if CAPA has a site search feature. If not currently searchable, omit the potentialAction property. This keeps schema accurate and honest.

BreadcrumbList Template Priority 2

Defines the navigation hierarchy on every page, helping AI engines understand site structure and categorization. When an AI engine reads that a page is under "Services > Cinema Advertising," it better understands context and relevance for user queries.

Applies to: All main pages (added to base template)

Where in HubSpot: Base template head or individual page templates; uses HubL variables to build breadcrumbs dynamically

JSON-LD Code (Example: Cinema Advertising page)
{

Build approach in HubSpot: Use HubL variables (content.name, content.parent_blog, URL structure) to dynamically generate breadcrumbs. Adjust position and item values based on actual site navigation.

FAQPage Module Module Priority 3

Captures question-and-answer content about CAPA's services in a structured format that AI engines can extract and cite directly. If someone asks ChatGPT "How much does cinema advertising cost?" or "What is product placement in film?", having FAQ schema with clear answers increases the likelihood CAPA is cited as the source.

Important: Only add FAQ schema for questions that are actually visible and answered on the page. Do not create schema for hidden Q&A. The FAQ content must be real and user-facing.

Note on content hours: Content hours apply only to FAQ writing. All other schema types (Organization, Service, WebSite, BreadcrumbList, AboutPage) pull content directly from what already exists on CAPA's site: company description, service names, team bios, page titles, social media URLs. No new content needs to be written for these. FAQ content hours assume AI generates a first draft of 5-8 Q&A pairs per service page (0.5-1h total), then a content writer reviews and edits for accuracy and brand voice. This is significantly faster than writing FAQ content from scratch. If CAPA already has FAQ sections on its service pages, these hours drop to zero--the developer just maps the existing content into the schema.
Note on dev hours: Estimates assume AI-assisted development using code generation and templates from this plan. Traditional implementation (writing schema from scratch without AI support) would take 1.5-2x longer. Template-level work (BreadcrumbList, Service schema) means a small up-front investment that automatically covers all pages of that type--for example, one BreadcrumbList implementation in the base template applies to 45+ pages across CAPA's site. Testing is included in the final 2-3 hours. Work can be split across multiple sprints if needed (e.g., Priority 1 in Sprint 1, Priority 2-3 in Sprint 2).
